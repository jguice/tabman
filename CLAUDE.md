# Tabman development

## Making and testing changes

1. Edit code in `src/` only. `workflow/` (untracked) and `Tabman.alfredworkflow` (tracked) are build artifacts regenerated by `./build.sh`; `src/` plus root `info.plist` are the single source of truth. Never `git add -f workflow/`.
2. Run `./build.sh` to regenerate `workflow/` and `Tabman.alfredworkflow`.
3. Reinstall the built package so Alfred runs exactly what would be released: `open Tabman.alfredworkflow`. Alfred prompts to import; the matching bundle id (`com.joshguice.tabman`) replaces the installed copy.
4. Test in Alfred (`tmt` / `tmb` / `tmh` keywords).

Never edit the installed copy under `Alfred.alfredpreferences/workflows/` directly; always go through build + reinstall so what gets tested is what gets pushed and released.

## Releases and versioning

Releases are fully automated; never bump the `info.plist` version by hand.
On push to `main`, `.github/workflows/release.yml` runs semantic-release:
commit messages determine the next version (conventional commits: `fix:` =
patch, `feat:` = minor, `BREAKING CHANGE` = major), CI patches the version
into `info.plist`, rebuilds, and attaches `Tabman.alfredworkflow` to the
GitHub release. Consequence: commit messages on branches headed to `main`
must use conventional-commit prefixes or the release math breaks.

## Testing scripts from the CLI

- Scripts must run with cwd = `src/` (they `eval` `lib_favicons.js` from the
  current directory): `cd src && /usr/bin/osascript -l JavaScript search_tabs.js 'query'`
- Always point tests at a throwaway cache, never the real Alfred cache:
  `export C=$(mktemp -d)` then pass `alfred_workflow_cache="$C"` in the env.
- Browser toggles arrive as env vars `enable_chrome|brave|arc|ghostty`
  ("1"/"0"); unset means enabled, so plain CLI runs behave all-on.
- For `tmt` tests that must not touch live browsers: seed
  `$C/tabs-snapshot.json` and pre-create `$C/rebuild.lock` (blocks the
  background rebuild from spawning).
- `search_tabs.js --rebuild` runs a full snapshot rebuild synchronously;
  runs that build snapshots send Apple Events to browsers and need the
  terminal's Automation permission (and window names in fingerprints need
  Screen Recording).

## Cache invariants (break these and toggles/staleness bugs return)

- tmb: the bookmark snapshot is keyed ONLY by source-file mtimes; disabled
  browsers must be dropped from `sources` before the fingerprint is
  computed, or stale rows serve forever.
- tmt: the window fingerprint carries an `|enabled:...` suffix (appended
  outside the try/catch) so toggling a checkbox invalidates the snapshot;
  `run()` also filters served rows by the enabled set for instant response.
- tmt: snapshots older than `sysctl kern.boottime` are discarded (window
  and Ghostty tab ids are reused across reboots). Do not switch this to
  `NSProcessInfo.systemUptime`; it excludes sleep and falsely expires
  same-boot snapshots.
- tmh: SELECT column order is `last_visit_time, title, url`; the URL must
  stay the last column (titles may contain tabs; the parser splits on the
  last tab) and rows sort globally by timestamp before the URL dedupe.

## Useful paths

- Installed workflow: `~/Library/Application Support/Alfred/Alfred.alfredpreferences/workflows/` (find it by grepping info.plist for `com.joshguice.tabman`)
- Workflow cache (snapshots, window shots, favicons): `~/Library/Caches/com.runningwithcrayons.Alfred/Workflow Data/com.joshguice.tabman/`
